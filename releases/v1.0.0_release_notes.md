# OpenFXpedia v1.0.0 — First stable release

## One-line summary
Initial stable release of OpenFXpedia — a currency converter and encyclopedia.

---

## Highlights
- Initial stable release of OpenFXpedia: currency converter + encyclopedia UI.
- Cross-platform builds with versioned artifacts for easy distribution.

## What’s new
- Settings UI:
  - Centralized Theme selector (System / Light / Dark).
  - App version display and in-app "Check updates" that compares against GitHub releases and prompts a device-specific download for `openfxpedia_<version>_setup.exe` on Windows or `openfxpedia_<version>.apk` on Android.
  - Changelogs fetched from GitHub release notes (paginated).
  - License page replacing the legacy About dialog (loads repository `LICENSE`).
- Packaging & build:
  - Build scripts produce versioned artifacts: `openfxpedia_<version>.apk`, `openfxpedia_<version>.exe`, and `openfxpedia_<version>_setup.exe`.
  - Windows release builds also produce an NSIS installer so `flutter_windows.dll` and the Flutter runtime are bundled for other PCs.
  - The installer is built with `makensis` and reuses the app icon from `windows/runner/resources/app_icon.ico`.
- Icons & assets:
  - Automatic icon generation via `tool/generate_icons_from_fiat.dart` (creates per-currency SVGs and `assets/icons/generic.svg`).
- Robust version handling:
  - Release tags with or without a leading `v` are supported (e.g., `v1.0.0` and `1.0.0` are treated equivalently).
  - Update checks ignore draft and prerelease GitHub releases.
- Documentation:
  - README updated with build instructions and artifact naming.

## Bug fixes & polish
- Replaced the about popup with a full License page for clearer attribution.
- Minor UI tidy-ups and accessibility improvements in Settings and changelog display.

## How to install / assets
Attach platform artifacts to this release (recommended filenames):
- `openfxpedia_1.0.0.apk`
- `openfxpedia_1.0.0.apk.sha1`
- `openfxpedia_1.0.0.exe`
- `openfxpedia_1.0.0_setup.exe`

The portable EXE is written to `build/windows/portable/` and the NSIS installer is written to `build/windows/installer/`.
The NSIS installer does not require MSIX certificates. If `makensis` is missing, install NSIS and ensure it is on PATH before running the Windows build script.

Users can check for updates in-app via Settings → App version → Check updates, then download the matching release asset for their device (`openfxpedia_<version>_setup.exe` on Windows or `openfxpedia_<version>.apk` on Android).

## Build commands (for maintainers)
Run the prepared scripts to build and produce versioned artifacts.

Bash (POSIX):

```bash
./scripts/build_android.sh
./scripts/build_windows.sh
```

PowerShell (Windows):

```powershell
./scripts/build_android.ps1
./scripts/build_windows.ps1
```

## Tagging & release notes
- Recommended tag format: `v1.0.0` (common practice). The app will also accept `1.0.0`.
- Paste this changelog into the GitHub release body; the app fetches full release notes from GitHub releases.
