# OpenFXpedia v1.0.2 — Maintenance release

## One-line summary
Switch the primary exchange-rate source to Frankfurter, improve localization, and polish startup UI.

---

## Highlights
- Frankfurter is now the primary exchange-rate API with a configurable fallback.
- Refactored rate source handling and updated `RateInfoWidget` integration.
- Improved localization across the app and added localized error messages.
- Added splash screen support with dark mode and localized assets.
- Removed an obsolete Flutter CI workflow configuration and updated the README.

## What’s new
- Exchange rates:
  - Use Frankfurter as the main exchange-rate provider; keep a fallback for reliability.
  - Simplified provider selection logic and improved error handling for network failures.
- Localization & assets:
  - Cleaned up i18n keys and enhanced translations for the encyclopedia and converter flows.
  - Localized error messages and added localized splash assets.
- UI polish:
  - Added a localized splash screen with dark-mode support.
- Maintenance:
  - Removed obsolete CI config and updated project documentation.

## Bug fixes & polish
- Fixed remaining untranslated UI surfaces in the encyclopedia and rate widgets.
- Improved resilience when the primary exchange service is unavailable.

## How to install / assets
Attach platform artifacts to this release (recommended filenames):
- `openfxpedia_1.0.2.apk`
- `openfxpedia_1.0.2.apk.sha1`
- `openfxpedia_1.0.2.exe`
- `openfxpedia_1.0.2_setup.exe`

Portable EXE: `build/windows/portable/`
NSIS installer: `build/windows/installer/` (requires `makensis` on PATH)

## Build commands (for maintainers)
Bash (POSIX):

```bash
./scripts/build_android.sh
./scripts/build_windows.sh
```

PowerShell (Windows):

```powershell
./scripts/build_android.ps1
./scripts/build_windows.ps1
```
